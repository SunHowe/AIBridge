AIBridge Workflow Graph 面板方案

用行为树风格的节点视图展示 AIBridge 工作流，但保持 AIBridge 自己的 workflow、Skill、recipe、gate 和 artifact 语义。

核心判断

推荐新增独立的 AIBridge/Workflow Graph 高级面板。 它用于可视化、受控编辑和生成 workflow，不替代现有 Skill 文档、recipe JSON 或 CLI 校验。

不是传统行为树 runtime 不做 Unity 每帧 tick，不承诺自动执行 LLM 或人工步骤。
是 Workflow Graph 节点表达 Preflight、分支、phase、step、gate、artifact 和 handoff。
先只读再编辑 第一版解决可理解性，后续再开放项目 recipe 编辑和生成预览。

面板边界

面板 职责 不负责
AIBridge/Workflows Skills、推荐库、Workflow Options、assistant 集成配置 recipe 编辑、run 调试、artifact 浏览、gate 排障
AIBridge/Workflow Graph routing、recipe、run 状态、gate 和 handoff 的图形展示与高级编辑 替代 CLI、自动执行 agent/manual、直接修改 installed Skill Markdown

关键约束：UI 不从 Markdown 猜测结构。Markdown 和 graph manifest 必须由同一生成器输出，避免双源漂移。

落地路线

1. Graph Viewer 只读展示 routing、recipe 和 active run 基础状态。
2. Run Evidence 展示 gate、artifact、external import gap 和 stale evidence。
3. Recipe Editor 只编辑项目本地 recipe，保存前 validate / plan。
4. Routing Manifest SkillInstaller 同源生成 Markdown 和 graph manifest。
5. Skill Preview 展示生成影响和 diff preview，再刷新 assistant Skills。

首版建议使用独立 IMGUI EditorWindow，降低对 experimental GraphView 的依赖，并保持模型与渲染解耦。

验收标准

  1. 文档和 UI 明确说明 Workflow Graph 不是传统 Behavior Tree runtime。
  2. AIBridge/Workflows 的默认用户配置定位保持不变。
  3. AIBridge/Workflow Graph 能只读展示 routing、recipe 和 run 状态。
  4. agent、manual、gate failed、artifact missing、stale evidence 都有明确视觉状态。
  5. UI 不直接编辑 installed Skill Markdown。
  6. 编辑功能只写项目 recipe 或 ProjectSettings。
  7. 所有 recipe 变更必须通过 workflow validate。
